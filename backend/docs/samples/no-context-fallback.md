# RAG 无上下文兜底

## 问题

RAG 系统不是只要能生成答案就算成功。如果知识库里没有相关资料，模型仍然强行回答，就容易出现幻觉。

例如用户问一个知识库没有收录的问题，如果系统直接调用模型，模型可能给出看似合理但没有依据的答案。

## 设计

DevMind 在调用模型前会先检索相关 chunk。

如果没有检索到有效 chunk，系统会返回无上下文兜底提示，而不是强行构造 Prompt 调用模型：

```text
知识库资料不足，无法回答该问题。
请先补充相关笔记，或换一个更明确的问题。
```

## 价值

- 减少模型幻觉。
- 避免无效 token 成本。
- 让用户知道需要补充知识库材料。
- 便于在 evaluation dataset 中验证负样例。

## 面试表达

检索不到 chunk 时，DevMind 不会强行让模型编答案，而是返回无上下文兜底。这是 RAG 幻觉控制的一部分，也能减少不必要的模型调用成本。
