# RAG 幻觉控制：无上下文兜底

## 什么是幻觉

大模型会生成**看似合理、实则没有依据**的内容。RAG（检索增强生成）的思路是：先从知识库检索相关资料，把资料塞进 Prompt 再让模型作答，用"有据可查"压制幻觉。

但 RAG 不是万能的：**如果知识库里根本没有相关资料，模型仍然会硬答**，反而披着"引用"的外衣更有迷惑性。

## 无上下文兜底

在调用模型前先看检索结果：**检索不到有效片段，就不构造 Prompt、不调模型，直接返回兜底提示**。

```text
知识库资料不足，无法回答该问题。
请先补充相关笔记，或换一个更明确的问题。
```

判断"检索不到"的常见做法：召回结果为空，或**最高相似度低于阈值**（低于阈值视为不相关，避免勉强命中）。

## 价值

- **减少幻觉**：没依据就明说没有，而不是编。
- **省成本**：跳过一次模型调用，不花无谓的 token。
- **可评估**：这类"知识库没有资料"的问题正好当**负样例**，验证系统会不会老实兜底，而不是乱答。

## 其他幻觉控制手段

- **Prompt 约束**：明确要求"只依据给定资料回答，资料没有就说不知道"。
- **强制引用**：让模型标注每句话来自哪个片段，答案与引用对不上就拦截。
- **相似度阈值 / rerank**：先把弱相关的片段过滤或降权，别喂进 Prompt。
- **事后校验**：用规则或另一个模型核对答案是否被检索内容支撑（faithfulness 校验）。
