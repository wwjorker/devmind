# Flyway Migration 数据库迁移

## 问题

项目早期可以手动在 DBeaver 执行 SQL 建表，但长期维护时容易出现不同环境表结构不一致的问题。例如本地加了字段，GitHub 或另一台电脑没有执行同一份 SQL，就会出现启动失败或字段缺失。

## 设计

DevMind 使用 Flyway migration 管理数据库结构变更。数据库脚本放在：

```text
src/main/resources/db/migration
```

脚本命名采用：

```text
V版本号__说明.sql
```

应用启动时，Flyway 会检查数据库当前版本，自动执行尚未执行过的迁移脚本。

## 价值

- 数据库结构和代码一起进入 Git 版本管理。
- 新环境只需要创建 `devmind` 数据库，启动项目后自动建表。
- 后续加表、加字段、加索引都有版本记录。
- 避免依赖人工复制 SQL，降低环境漂移风险。

## 面试表达

Flyway 解决的是数据库结构版本管理问题。它把手动 SQL 变成版本化 migration，让应用启动时自动迁移数据库，避免不同环境表结构不一致。
